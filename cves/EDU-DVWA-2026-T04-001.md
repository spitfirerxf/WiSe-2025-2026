**CVE-ID**: EDU-DVWA-2026-T04-001
**Title**:  Improper Authentication through Default Credentials
**Affected Lab**:  DVWA (VMware Metasploitable OR `10.6.6.13`)
**Component**:  `/login.php`
**Severity**:  Critical
**CVSS Vector**:  `CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H`
**CVSS Score**:  9.8
**Description**:  The application is deployed with a well-known, default administrative account. The login mechanism does not require a password change upon initial setup, allowing any remote attacker with network access to the application to gain full administrative privileges by using the default credentials `admin:password`

Payload:
Username: `admin`
Password: `password`

**Steps to Reproduce**:
1. Navigate to Target: Open a web browser and go to the DVWA login interface: `http://10.6.6.11/login.php`.
2. Input Credentials: In the "Username" field, type `admin`. In the "Password" field, type `password`.
3. Execute Login: Click the Login button to submit the authentication request.
4. Verify Access: Observe that the server accepts the credentials and redirects the browser to the main dashboard (`index.php`), granting full access to the application's administrative features.
5. Confirm Identity: (Optional) Check the "UserID" or "Welcome" section of the page to confirm you are logged in as the administrative user.

**Remediation**:

1. **Immediate Action**: Change the default password for the admin user immediately via the "User Management" or "Setup" page.
2. **Enforce Password Complexity**: Implement a policy that requires a strong, unique and complex password during the initial setup process.
3. **Account Lockout**: Implement account lockout mechanisms to prevent brute-force attempts on administrative accounts.
